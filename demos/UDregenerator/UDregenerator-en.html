Universal Dependencies graph/tree display with regeneration using jsRealB

[Version française]
  • Copy one or many dependency structures in the text box in CONLLU format
    A local file can also be loaded by clicking on the button under the text field.
  • Click on the Parse button to build the menu. If Only different is checked, then only sentences whose realisation is different from the original text appear in the menu.
  • If the checkbox Allow out of vocabulary words is checked, then jsRealB will consider them as Q(...) and they will be copied verbatim in the output. Unfortunately the display in the editor shows the original constructors because this modification is handled at the level of jsRealB. This flag can be convenient but it should be used sparingly. Parse must be clicked so that this option takes effect.
  • Select a sentence from the menu. A starting asterisk indicates a non-projective dependency structure.
  • The graph or tree of dependencies will be shown. The root word is shown in red. By hovering over a word, infos about this words will appear in a tooltip.
  • An English sentence will be regenerated from the dependencies by jsRealB
  • A table shows the edit operations to transform the re generated sentence to the original text of the UD: replacement, insertion, deletion. Differences in spacing are ignored.
  • The jsRealB expression appears in a text area allowing its edition for re-realization by clicking on the Realize button.
